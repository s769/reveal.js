Optimal Experimental Design (OED)

  • Initial candidate set: $N_d = 600$ sensors.
  • Cost constraints limit realistic deployment to ~150-200 sensors.
Goal

Select the optimal subset $S$ ($|S| \leq B$) that provides maximum information about the uncertain parameters $\mathbf{m}$ or the QoI $\mathbf{q}$.

Notation:

  • Subscript $S$ (e.g., $\mathbf{F}_S, (\Gamma_{\!\text{post}})_S$) restricts to the sensors in $S$.
  • For $\mathbf{F}_S$, $\mathbf{K}_S$, etc. $\Rightarrow$ select the associated block rows/columns.
  • Other operators such as $(\Gamma_{\!\text{post}})_S$ and $(K^{-1})_S$ must be computed using their definitions, restricting to the sensors in $S$.

Expected Information Gain & D-Optimality

  • Objective: Maximize expected information gain from prior to posterior.
  • For linear-Gaussian inverse problems, this equals the D-optimal design criterion.
  • We evaluate two criteria:
    • Standard: Uses parameter posterior covariance $(\Gamma_{\!\text{post}})_S$
    • Goal-oriented: Uses QoI posterior covariance $(\Gamma_{\!\text{post}(q)})_S$

Standard D-optimal criterion: Minimize log-determinant of posterior covariance.

\[ S_{\text{opt}} = \arg\min_{S, |S| \le B} \log\det((\Gamma_{\!\text{post}})_S) \]

Standard D-optimal Design

Use the matrix determinant lemma for a computable form:

\[\footnotesize \begin{align*} \det((\Gamma_{\!\text{post}})_S) & = \det\left(\Gamma_{\!\text{prior}} - \mathbf{G}_S^*\mathbf{K}_S^{-1}\mathbf{G}_S\right) \\ & = \det(\Gamma_{\!\text{prior}})\det\left(\mathbf{I} - \Gamma_{\!\text{prior}}^{-1}\mathbf{G}_S^*\mathbf{K}_S^{-1}\mathbf{G}_S\right) \\ & = \det(\Gamma_{\!\text{prior}})\det\left(\mathbf{I} - \mathbf{K}_S^{-1}\mathbf{G}_S\Gamma_{\!\text{prior}}^{-1}\mathbf{G}_S^*\right) \\ & = \det(\Gamma_{\!\text{prior}})\det(\mathbf{K}_S^{-1})\det\left(\mathbf{K}_S - \mathbf{G}_S\Gamma_{\!\text{prior}}^{-1}\mathbf{G}_S^*\right) \\ & = \det(\Gamma_{\!\text{prior}})\det(\mathbf{K}_S^{-1})\det\left((\Gamma_{\!\text{noise}})_S\right) \end{align*} \]
Computable Objective Function

\[ S_{\text{opt}} = \arg\max_{S, |S| \le B} \log\det(\mathbf{K}_S) \]

Assuming $\Gamma_{\text{noise}} = \gamma^2\mathbf{I}$

Standard D-optimal Design: Greedy Algorithm

  • Combinatorial optimization is NP-hard $\rightarrow$ use a greedy approximation.

Algorithm Outline:

  1. Initialize $S = \emptyset$.
  2. For $k = 1$ to $B$,
    find $i^* = \arg\max_{i \notin S} \left[ \log\det(\mathbf{K}_{S \cup \{i\}}) - \log\det(\mathbf{K}_S) \right]$
  3. Add best sensor: $S \leftarrow S \cup \{i^*\}$.

Efficient Computation:

  • Use Schur complement: $\text{Gain}(i) = \log \det(\mathbf{K}_{ii} - \mathbf{K}_{iS} \mathbf{K}_S^{-1} \mathbf{K}_{Si})$
  • Avoid $\mathbf{K}_S^{-1}$ via Cholesky updates ($\mathbf{K}_S=\mathbf{L}_S \mathbf{L}_S^\top$).
  • Evaluating candidate sensors $i$ is embarrassingly parallel.

Goal-oriented D-optimal Design

Criterion: $S_{\text{opt}} = \arg\min_{S, |S| \le B} \log\det((\Gamma_{\!\text{post}(q)})_S)$

Matrix determinant lemma yields the computable form:

\[\footnotesize \begin{align*} &\det((\Gamma_{\!\text{post}(q)})_S)\\ = &\det\left(\mathbf{F}_q \mathbf{G}_q^* - \mathbf{F}_q \mathbf{G}_S^* \mathbf{K}_S^{-1} \mathbf{G}_S \mathbf{F}_q^*\right) \\ = &\det(\mathbf{F}_q \mathbf{G}_q^*) \det\left(\mathbf{I} - (\mathbf{G}_S \mathbf{F}_q^*) (\mathbf{F}_q \mathbf{G}_q^*)^{-1} (\mathbf{F}_q \mathbf{G}_S^*) \mathbf{K}_S^{-1}\right) \\ = &\det(\mathbf{F}_q \mathbf{G}_q^*) \det\left(\mathbf{I} - (\mathbf{F}_q \mathbf{G}_q^*)^{-1} \left[ (\mathbf{F}_q \mathbf{G}_S^*) \mathbf{K}_S^{-1} (\mathbf{G}_S \mathbf{F}_q^*) \right]\right) \end{align*} \]
Computable Objective Function

\[\small S_{\text{opt}} = \arg\max_{S, |S| \le B} -\log \det\left(\mathbf{I} - (\mathbf{F}_q \mathbf{G}_q^*)^{-1} \left[ (\mathbf{F}_q \mathbf{G}_S^*) \mathbf{K}_S^{-1} (\mathbf{G}_S \mathbf{F}_q^*) \right] \right) \]

Goal-oriented D-optimal Design: Algorithm

Uses the identical greedy iterative structure, but maximizes the
goal-oriented objective $J_g(S)$.

Efficient Score Calculation (for candidate $i$):

  1. Compute Cholesky factor $\mathbf{L}_{S \cup \{i\}}$ via fast Schur update.
  2. Construct $\mathbf{Y}_{S \cup \{i\}} = \mathbf{G}_{S \cup \{i\}} \mathbf{F}_q^*$.
  3. Solve $\mathbf{K}_{S \cup \{i\}} \mathbf{X} = \mathbf{Y}_{S \cup \{i\}}$ for $\mathbf{X}$.
  4. Compute core matrix $\mathbf{M}_{\text{core}} = \mathbf{Y}_{S \cup \{i\}}^* \mathbf{X}$.
  5. Score $= -\log\det(\mathbf{I} - (\mathbf{F}_q \mathbf{G}_q^*)^{-1} \mathbf{M}_{\text{core}})$.

Note: Computationally heavier than standard case, but effectively parallelizable.

Greedy Algorithm: Key Properties

  • Precomputation: All relevant operators ($\mathbf{K}, (\mathbf{F}_q\mathbf{G}_q^*)^{-1}, \mathbf{F}_q\mathbf{G}^*$) are precomputed/stored as dense matrices via fast FFT-based algorithms.1,2
  • Submodularity: Both objectives exhibit submodularity (diminishing returns).
  • Theoretical Guarantee: Greedy selection achieves at least a
    $(1 - 1/e)$-approximation ($\sim$63%) of the true optimal solution.

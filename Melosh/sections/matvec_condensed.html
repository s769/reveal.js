Fast p2o Map Applications

  • There is a need for fast Hessian applications to enable real-time inference and prediction
  • Most expensive part of Hessian application is the p2o operator (usually applied via forward/adjoint PDE solves)
  • Autonomous Dynamical System: evolution does not depend explicitly on independent variable (e.g. time)
    • Here: the mapping mapping \(m(x,t+\tau) \mapsto d(x,t+\tau)\) is the same as the mapping \(m(x,t) \mapsto d(x,t)\)
  • Exploit (time) shift invariance of autonomous dynamical systems
    • Reduce computation and storage costs

FFTMatvec Overview

  • FFT-based, GPU-accelerated algorithm for fast p2o applications
    1. Broadcast and add zero-padding to input vector \(\mathbf{m}\)
    2. Compute FFT of input vector \(\mathbf{m}\mapsto\hat{\mathbf{m}}\)
    3. Compute block-diagonal Fourier space matvec \(\hat{\mathbf{m}}\mapsto\hat{\mathbf{d}}\)
    4. Compute IFFT of output vector \(\hat{\mathbf{d}}\mapsto\mathbf{d}\)
    5. Remove padding and compute reduction on output vector \(\mathbf{d}\)
  • Same algorithm for \(\mathbf{F}^*: \mathbf{d} \mapsto \mathbf{m}\)
    • Conjugate transpose SBGEMV in Step 3
    • No extra storage or communication needed
  • Cost: \(O(N_dN_mN_t + (N_m+N_d)N_t\log N_t)\) (for single GPU)
  • Algorithm is exact and provides \(\sim 200,000\times\) speedups over PDE solve method for p2o applications
  • Open-source, performance-portable, mixed-precision implementation on GitHub
